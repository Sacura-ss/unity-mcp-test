# Создание и назначение AssetBundle

## Инструкция для агента

### 1. Определение целевой папки
- Спроси пользователя: **«Для какой папки вы хотите создать AssetBundle?»** (например, `Art/Environment`, `Art/UI` или `Art`).
- Если папка не существует – создай её через MCP-инструмент `create_folder` (или предложи пользователю создать вручную, если MCP не поддерживает).

### 2. Назначение основного бандла
- Спроси имя бандла (по умолчанию предложи `{имя_папки}_bundle`, например `environment_bundle`).
- **Важно:** назначь этот бандл **на каждый файл** в целевой папке (не рекурсивно – только файлы первого уровня). Используй MCP-инструмент `manage_asset` с действием `modify` и свойством `assetBundleName`, установив указанное имя.
- **Как получить список файлов:** используй MCP-инструмент `list_directory` (или `get_assets_in_folder`), чтобы получить пути всех файлов внутри целевой папки (игнорируя подпапки). Для каждого файла выполни `manage_asset`.
- Если в папке нет файлов, сообщи об этом пользователю и предложи импортировать ассеты сначала (через `/import-art`).

### 3. Обработка вложенных папок (переопределение)
- После назначения основного бандла проверь, есть ли внутри целевой папки **вложенные подпапки** (через `list_directory` – получи только папки).
- **Если вложенные папки есть**:
  - Спроси пользователя:  
    > «В папке `{папка}` есть вложенные папки: `{список}`. Хотите ли вы для каких‑либо из них назначить **отдельные** бандлы (чтобы они не наследовали родительский)?»
  - Если пользователь отвечает «да»:
    - Предложи выбрать конкретные папки из списка (можно ввести названия через запятую) или выбрать опцию **«все»**.
    - **Если выбрана опция «все»**:
      - Спроси пользователя, хочет ли он **задать имена бандлов вручную** для каждой подпапки или **использовать автоматические** имена по умолчанию (`{имя_подпапки}_bundle`).
      - При автоматическом режиме — сразу назначь для каждой подпапки бандл с именем по умолчанию (рекурсивно на все файлы внутри).
      - При ручном режиме — для каждой подпапки запроси имя бандла (с предложением значения по умолчанию) и назначь его.
    - **Если выбраны конкретные папки** (перечислены через запятую):
      - Для каждой выбранной вложенной папки:
        - Спроси имя бандла для этой папки (по умолчанию `{имя_подпапки}_bundle`).
        - Назначь этот бандл **на каждый файл внутри этой подпапки рекурсивно** (т.е. на все файлы во всех уровнях вложенности). Для этого:
          - Получи список всех файлов внутри этой подпапки (рекурсивно) через `list_directory` с опцией `recursive: true`.
          - Для каждого файла используй `manage_asset` с действием `modify` и свойством `assetBundleName`, установив новое имя бандла.
  - Если пользователь отвечает «нет» – пропусти этот шаг. Тогда все файлы во вложенных папках **останутся с бандлом, назначенным на родительскую папку** (если они его наследуют), либо будут без бандла, если вы не назначали на них ранее.

### 4. Проверка и завершение назначений
- Сообщи пользователю краткую сводку:
  - Какие бандлы и на какие папки (или файлы) назначены.
  - Какие вложенные папки получили собственные бандлы (если были).

### 5. Предложение выполнить сборку
- Спроси пользователя:  
  > «Все назначения выполнены. Хотите ли вы **собрать** все AssetBundle сейчас (в папку `BuiltBundles`)?»
- Если «да»:
  - Проверь, существует ли папка `BuiltBundles` в корне проекта (если нет – создай её через MCP `create_folder`).
  - Выполни сборку через MCP-инструмент `build_asset_bundles` (убедись, что он принимает путь `BuiltBundles`).
  - После сборки сообщи пользователю пути к файлам и их размеры (если MCP может предоставить информацию о файлах – используй `get_file_info`, иначе скажи, где смотреть и предложи открыть папку).
- Если «нет»:
  - Сообщи, что сборку можно будет выполнить позже, вызвав отдельный скилл `/build-bundles` (если он есть) или вручную через меню Unity (Window → AssetBundle Browser → Build).

### 6. Дополнительные замечания
- Если MCP-инструмент `assign_asset_bundle` поддерживает назначение на папку целиком, вы можете использовать его для ускорения, но в данном скилле мы предполагаем пофайловое назначение через `manage_asset` для максимальной гибкости.
- Если папка уже имеет какие-то бандлы, уточни у пользователя, нужно ли их перезаписывать (по умолчанию – перезаписываем, но можно спросить).
- В случае ошибок (например, файл не найден или `manage_asset` недоступен) – сообщи пользователю и предложи продолжить или прервать.
