# Создание и назначение AssetBundle

## Инструкция для агента

### 1. Определение целевой папки
- Спроси пользователя: **«Для какой папки вы хотите создать AssetBundle?»** (например, `Art/Environment`, `Art/UI` или `Art`).
- Если папка не существует – создай её через MCP-инструмент `create_folder` (если поддерживается).

### 2. Определение имени бандла
- Спроси пользователя: **«Какое имя вы хотите дать этому бандлу?»**.
- Предложи дефолтное имя: **`{последняя_часть_пути}_bundle`** (например, для `Art/Environment` – `environment_bundle`; для `Art` – `art_bundle`).
- Запомни выбранное имя (назовём его **`baseName`**).

### 3. Определение пропускаемых вложенных папок
- С помощью MCP-инструмента `list_directory` с опцией `recursive: false` получи список **подпапок** внутри целевой папки.
- Если вложенные папки есть, спроси пользователя:
  > «В папке `{целевая_папка}` есть следующие вложенные папки: `{список}`. Какие из них вы хотите **пропустить** (т.е. не включать в этот бандл)? Укажите имена через запятую или введите `0`, если пропускать ничего не нужно.»
- Если пользователь вводит список имён – это **исключаемые подпапки**. Все файлы внутри них **не получат** данный бандл (бандл для них будет сброшен, если они унаследовали его от родительской папки).
- Если пользователь вводит `0` – ничего не пропускаем (все подпапки и их содержимое получат бандл).

### 4. Определение типа размещения (корневой / вложенный)
- Спроси пользователя:
  > «Как вы хотите разместить этот бандл в папке `BuiltBundles`?»
  - **Корневой** – файл бандла будет лежать прямо в `BuiltBundles` (имя без слешей, например `environment_bundle`).
  - **Вложенный** – файл бандла будет лежать внутри подпапки (имя со слешами, например `Environment/environment_bundle`).
- Если пользователь выбрал **вложенный**, спроси, **какой путь** добавить перед именем (без ведущего слеша). По умолчанию предложи имя самой папки (например, для папки `Art/Environment` предложи `Environment`). Итоговое полное имя бандла будет `{путь}/{baseName}`.
- Если выбран корневой – полное имя бандла будет просто `{baseName}`.

### 5. Назначение бандла на папку и исключение пропущенных
- **Предпочтительный способ** – назначить бандл **на саму целевую папку** через `manage_asset` с действием `modify` и свойством `assetBundleName`, установив **полное имя бандла**. Это автоматически применится ко всем файлам внутри папки (рекурсивно), за исключением тех, для которых бандл переопределён явно.
- Если MCP-инструмент не поддерживает назначение на папки, тогда получи список **всех файлов** внутри целевой папки (рекурсивно, кроме пропущенных подпапок) и назначь бандл каждому файлу отдельно. Но сначала попробуй назначить на папку – это эффективнее.

- **Для пропущенных подпапок** (из шага 3) необходимо **сбросить** бандл, чтобы они не унаследовали его от родительской папки. Для этого:
  - Для каждой пропущенной подпапки (и всех её файлов, если они уже получили бандл от родителя) установи `assetBundleName` в пустую строку (или `null`) с помощью `manage_asset`.
  - Если бандл назначался на файлы индивидуально, то просто не назначай его на файлы из пропущенных папок (они останутся без изменений).

### 6. Проверка и сводка
- После выполнения всех операций выведи краткую сводку:
  - Полное имя бандла: `{полное_имя}`.
  - Количество файлов, получивших бандл (если возможно подсчитать).
  - Пропущенные подпапки (если были).
- Сообщи пользователю, что назначение выполнено.

### 7. Рекомендация по сборке
- После завершения назначений **всегда** напомни пользователю:
  > «Назначения завершены. Для сборки AssetBundle в папку `BuiltBundles` выполните команду `/build-bundles`.»
- Не предлагай выполнить сборку сейчас через MCP – только эту команду.
